Accessibility Key Concepts

The W3C Web Accessibility Initiative (WAI) develops standards and support materials to help you understand and implement accessibility.

  1. Perceivable by at least one sense
  2. Operable, using keyboard, mouse, voice...
  3. Understandable, it should make sense to the users
  4. Robust, content should use web standards compatible with different browsers, currently and in the foreseeable future

  • Website Accessibility Conformance Evaluation Methodology (WCAG-EM) is an approach for determining how well a website conforms to Web Content Accessibility Guidelines.

  • WCAG-EM applies to all websites, including web applications and mobile websites. It covers different situations, including self-assessment and third-party evaluation. It is independent of particular evaluation tools, web browsers, or assistive technologies.

  • WCAG-EM is primarily for evaluating conformance of existing websites. However, accessibility should not be left until the evaluation stage; it should be integrated from the beginning and throughout the project lifecycle â€” in planning, design, and development.

Semantic HTML tags provide meaning to the content they enclose, making it more accessible to users and search engines. Screen readers and other assistive technologies rely on semantic structure to interpret and present content effectively to users with disabilities.

Accessible Rich Internet Applications (ARIA) enhances the accessibility of web content and applications for users with disabilities. It provides additional semantics to HTML elements, making dynamic content and interactive controls more understandable and navigable for assistive technologies.
